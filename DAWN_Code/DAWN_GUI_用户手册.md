## 概述

DAWN GUI 是一个基于深度学习的视频去噪和增强图形用户界面软件。它支持单通道和双通道（荧光显微镜）视频处理，目前支持 DAWN: A Spatiotemporal Self-supervised Framework for Reconstructing Ultra–Low-Photon Single-Molecule Fluorescence Images 和 Supervised multi-frame dual-channel denoising enables long-term single-molecule FRET under extremely low photon budget提出的MUFFLE模型

### 核心功能
- **单通道模式**：处理单通道灰度视频
- **双通道模式**：处理双通道荧光视频（如 Acceptor 和 Donor 通道）
- **GPU 加速**：利用支持 CUDA 的 GPU 进行快速处理
- **实时预览**：逐帧浏览输入和输出视频
- **系统监控**：处理过程中监控 CPU、GPU 和内存使用情况

## 功能特性

### 1. 双处理模式
- **单通道模式**：用于标准灰度视频去噪
- **双通道模式**：用于双通道荧光显微镜（Acceptor/Donor）

### 2. 灵活的模型加载
- 支持 `.pth`(DAWN) 和 `.pkl`(MUFFLE) 模型文件
- 独立的 YAML 配置文件选择
- 加载模型时自动检测 YAML 文件

### 3. 输入/输出管理
- 拖放加载视频文件
- 支持输入文件/文件夹路径进行批量推理
- 支持 TIFF 序列
- 自动创建带时间戳的输出文件夹
- 处理完成后快速访问输出文件夹

### 4. 处理控制
- **线性增益**：调整输入信号强度
- **设备选择**：选择 GPU 或 CPU 进行处理
- **进度跟踪**：实时进度条
- **中断处理**：点击“Stop”可中断当前推理

### 5. 可视化
- 逐帧视频预览
- 滑块控制导航
- 输入/输出并排比较
- 自动图像填充为正方形显示

### 6. 系统监控
- CPU 使用率和内存统计
- GPU 利用率和内存跟踪
- 温度和功耗监控（如果支持）
- 处理过程中实时更新

## 系统要求

### 最低要求
- **操作系统**：Windows 10/11、Linux 或 macOS
- **Python**：3.7 或更高版本
- **内存**：8 GB（推荐 16 GB）
- **存储空间**：应用程序 500 MB，视频文件需要额外空间

### GPU 加速要求（推荐）
- **GPU**：支持 CUDA 的 NVIDIA GPU
- **CUDA**：10.2 或更高版本
- **cuDNN**：与 CUDA 版本兼容的版本
- **显存**：最少 4 GB（大视频推荐 8 GB）

## 快速开始

### 快速入门指南

1. **启动应用程序**：

2. **选择处理模式**：
   - 使用单选按钮选择"单通道"或"双通道"

3. **加载输入视频**：
  - **拖放**视频文件到输入查看器上，或在输入框中填写文件/文件夹路径后回车
  - 支持格式：TIFF 序列文件
  - 输入文件夹时会按文件名排序进行批量推理（双通道需保证两侧文件数量一致）

4. **加载模型文件**：
   - 点击"Browse model file"选择 `.pth`(DAWN) 和 `.pkl`(MUFFLE) 模型文件
   - 点击"Browse Config file (.yaml)"选择 YAML 配置文件
   - 模型和配置文件应匹配（例如，一起训练的）

5. **配置设置**：
   - 如需要，设置线性增益（默认：1.0）
   - 选择 GPU 设备（推荐）或 CPU
   - 选择输出文件夹（默认：`./outputs`）

6. **开始处理**：
   - 点击"Reconstruct"按钮
   - 在进度条中查看进度
   - 在系统监控中查看系统资源

7. **查看结果**：
   - 处理后的视频显示在输出查看器中
   - 使用滑块浏览帧
   - 点击"Open folder"访问保存的输出文件

## 详细使用说明

### 1. 加载输入视频

#### 拖放方法
1. 找到视频文件（TIFF 序列）
2. 从文件资源管理器中拖拽文件
3. 放置到相应的输入查看器上：
   - **双通道模式**：放置到"Acceptor dark"或"Donor dark"查看器上
   - **单通道模式**：放置到"Single channel input"查看器上
4. 视频将自动加载并显示第一帧

#### 视频格式要求
- **格式**：TIFF 序列文件（`.tif` 或 `.tiff`）
- **结构**：多页 TIFF 或 TIFF 文件序列
- **数据类型**：8 位、16 位或 32 位灰度
- **尺寸**：任意大小（高度 × 宽度），帧数可变化

### 2. 模型和配置文件

#### 模型文件（.pth 或 .pkl）
- 包含训练好的神经网络权重
- 必须与所选处理模式匹配（单通道/双通道）
- 文件扩展名：`.pth`(DAWN) 和 `.pkl`(MUFFLE) 模型文件

#### 配置文件（.yaml）
- 包含模型架构参数
- 必须与模型文件对应
- 文件扩展名：`.yaml` 或 `.yml`
- 格式：YAML/OmegaConf 配置

#### 加载模型文件
1. **手动选择**：
   - 点击"Browse model file"按钮
   - 导航到模型文件位置
   - 选择文件并确认
   - 如需要，重复操作选择 YAML 配置文件

2. **自动检测**：
   - 加载模型文件时，GUI 会自动搜索匹配的 YAML 文件
   - 如果在同一目录中找到同名 YAML 文件，将自动选择
   - 您仍可以手动更改 YAML 文件

3. **拖放**（用于模型文件）：
   - 将模型文件拖放到模型路径显示区域
   - 如果可用，将自动检测 YAML

### 3. 处理设置

#### 线性增益
- **用途**：在处理前乘以输入信号强度
- **范围**：正浮点数（通常为 0.1 到 10.0）
- **默认值**：1.0（无缩放）
- **用法**：
  - 建议用于训练集和测试集亮度不匹配的情况，若亮度匹配不需要调整
  - 更改值后点击"Set Linear Gain"

#### 模式选择
- **单通道**：处理一个输入视频
  - 使用 DBSN_centrblind 模型架构
  - 单输入 → 单输出
  
- **双通道**：处理两个相关视频
  - 使用 DBSN_fusion 模型架构或者MUFFLE模型
  - 两个输入（Acceptor + Donor）→ 两个输出

#### 设备选择
- **GPU（推荐）**：
  - 需要支持 CUDA 的 NVIDIA GPU
  - 从设备列表中选择可用 GPU
  - 格式："GPU 名称 (cuda:0)"、"GPU 名称 (cuda:1)" 等

- **CPU**：
  - 较慢但通用

### 4. 输出配置

#### 输出文件夹
- **默认**：`./outputs`
- **自定义位置**：点击"Browse Output Folder"选择
- **自动组织**：输出保存在带时间戳的子文件夹中
- **格式**：`DAWN_result_YYYYMMDD_HHMMSS/`

#### 输出文件
- **单通道模式**：
  - `<输入文件名>_enhanced.tif` - 增强的视频序列

- **双通道模式**：
  - `<Acceptor文件名>_enhanced.tif` - 增强的 Acceptor 通道
  - `<Donor文件名>_enhanced.tif` - 增强的 Donor 通道
  - 批量推理会为每个输入文件生成对应输出

- **文件格式**：16 位 TIFF 序列
- **数据范围**：0-65535（从归一化的 0-1 缩放）

### 5. 视频导航

#### 帧滑块
- 位于每个视频查看器下方
- 拖拽以浏览视频帧
- **输入视频**：显示应用线性增益的原始数据
- **输出视频**：显示处理/去噪后的结果

#### 键盘快捷键
- 目前，导航主要通过滑块的鼠标操作

### 6. 系统监控

#### CPU 监控
- **CPU 使用率**：CPU 利用率百分比
- **内存使用**：百分比和绝对值（GB）
- 每 0.5 秒更新一次

#### GPU 监控（如果可用）
- **GPU 利用率**：GPU 计算使用百分比
- **内存使用**：百分比和绝对值（GB）
- **温度**：GPU 温度（摄氏度，如果支持）
- **功耗**：功耗（瓦特，如果支持）
- 多个 GPU 分别列出


## 故障排除

### 常见问题

#### 1. "GPU is not available" 警告
**问题**：未检测到 CUDA/GPU  
**解决方案**：
- 验证 CUDA 安装：`nvcc --version`
- 检查 PyTorch CUDA 支持：`python -c "import torch; print(torch.cuda.is_available())"`
- 如需要，重新安装支持 CUDA 的 PyTorch
- 应用程序将回退到 CPU（速度较慢）

#### 2. "Model file does not exist" 错误
**问题**：模型文件路径无效或缺失  
**解决方案**：
- 验证文件路径是否正确
- 检查文件权限
- 确保文件扩展名为 `.pth` 或 `.pkl`
- 尝试使用绝对路径而不是相对路径

#### 3. "Configuration file does not exist" 错误
**问题**：YAML 文件缺失或错误  
**解决方案**：
- 确保 YAML 文件与模型文件匹配
- 检查 YAML 文件语法（有效的 YAML 格式）
- 验证文件路径可访问
- 尝试手动选择配置文件

#### 4. "Please load input video(s) first" 错误
**问题**：未加载输入视频  
**解决方案**：
- 将视频文件拖放到输入查看器上
- 验证视频文件为 TIFF 格式
- 检查文件是否损坏
- 确保文件路径可访问

#### 5. 处理速度非常慢
**问题**：使用 CPU 或设置效率低  
**解决方案**：
- 如果可用，切换到 GPU（速度快得多）
- 如果可能，降低视频分辨率
- 关闭其他占用 GPU 的应用程序
- 检查系统监控以识别资源瓶颈

#### 6. 输出视频全黑/全白
**问题**：模型未正确加载或输入范围问题  
**解决方案**：
- 验证模型文件与所选模式匹配（单通道/双通道）
- 检查模型和配置文件是否兼容
- 调整线性增益值
- 验证输入视频数据范围（应为 0-1 或 0-255）

#### 7. 内存错误
**问题**：GPU/CPU 内存不足  
**解决方案**：
- 降低视频分辨率
- 处理较短的视频序列
- 关闭其他应用程序
- 如果 GPU 内存是问题，使用 CPU
- 如果持续出现，升级系统 RAM/VRAM

#### 8. 应用程序崩溃
**问题**：各种潜在原因  
**解决方案**：
- 检查控制台/终端的错误消息
- 验证所有依赖项是否正确安装
- 更新 PyTorch 和 CUDA 到最新兼容版本
- 检查系统日志以获取详细错误信息
- 首先尝试处理较小的测试视频

### 性能提示

1. **使用 GPU**：始终优先使用 GPU 而不是 CPU
2. **批处理**：顺序处理多个视频，而不是同时处理
3. **最佳视频大小**：根据需求平衡分辨率和处理时间
4. **监控资源**：使用系统监控识别瓶颈
5. **关闭不必要的应用**：处理前释放 GPU/CPU 资源

### 获取帮助

如果遇到此处未涵盖的问题：
1. 检查控制台/终端输出中的错误消息
2. 验证所有文件（视频、模型、配置）是否有效
3. 首先使用示例/测试数据进行测试
4. 查阅模型训练文档以了解模型特定要求
5. 检查系统要求和兼容性
